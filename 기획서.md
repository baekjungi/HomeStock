# 홈스톡(HomeStock) 서비스 기획서

## 1. 서비스 개요
**서비스명:** 홈스톡(HomeStock)  
**한 줄 소개:** 집에 있는 생필품 재고를 기록하고, 사용 패턴을 바탕으로 소진 시점을 예측해 장보기까지 연결해주는 생활 밀착형 서비스

---

## 2. 기획 배경과 문제 정의
### 왜 필요한가
많은 가정에서 생필품을 체계적으로 관리하지 못해 다음 문제가 반복됩니다.

1. 다 떨어진 뒤에야 구매함
2. 필요한 물건을 까먹고 마트에서 빠뜨림
3. 집에 있는데 또 사는 중복 구매 발생
4. 가족 간 재고 정보가 공유되지 않아 비효율 발생

### 문제의 본질
기존 앱은 현재 수량 기록 중심이라, 실제 생활에서는

- 언제 떨어지는지 모르고
- 언제 사야 하는지 판단이 어렵고
- 장보기 행동까지 자연스럽게 이어지지 않습니다.

---

## 3. 서비스 목표
홈스톡은 사용자가 생필품을 미리 관리하게 만들어

1. 부족 사태를 줄이고
2. 중복 구매를 방지하고
3. 계획적인 소비 습관을 형성하도록 돕습니다.

---

## 4. 핵심 가치 제안
### 사용자에게 주는 가치
1. 기억에 의존하지 않아도 됨
2. 떨어지기 전에 알림을 받아 선제 대응 가능
3. 장보기 리스트가 자동 생성되어 실행이 쉬움
4. 가족과 같은 재고를 보며 함께 관리 가능

### 공모전 차별 포인트
단순 재고 기록 앱이 아니라,
**생활 패턴 기반 소진 예측 + 장보기 자동화 + 가족 협업**을 하나로 묶은 서비스입니다.

---

## 5. 주요 기능
### 5-1. 생필품 등록
- 직접 입력
- 바코드 스캔
- 최근 구매내역 불러오기(선택)

대상 품목 예시: 휴지, 샴푸, 치약, 세제, 종량제 봉투, 섬유유연제

### 5-2. 재고 관리
- 품목별 수량을 직관적으로 입력
- 단위 다양화 지원
  - 개수형: 치약 2개
  - 퍼센트형: 샴푸 45%
  - 롤형: 휴지 6롤

### 5-3. 사용 기록 및 패턴 학습
- 사용자가 간단 버튼만 누르면 기록
  - 예: 휴지 1롤 사용
- 앱이 평균 사용 주기를 계산
  - 예: 휴지 5일당 1롤

### 5-4. 예상 소진일 계산
- 현재 재고와 평균 사용량을 조합해
  - 남은 사용 가능 일수
  - 예상 소진 날짜 자동 표시

### 5-5. 사전 알림
- 소진 7일 전, 3일 전, 1일 전 푸시 알림
- 예: 휴지가 3일 후 소진 예정입니다

### 5-6. 장보기 리스트 자동 생성
- 소진 임박 품목 자동 체크리스트 추가
- 사용자는 구매 후 체크만 하면 재고 자동 반영

### 5-7. 가족 공유
- 구성원 초대 기능
- 누가 구매/사용해도 실시간 재고 동기화

### 5-8. 확장 기능
- 집에 있어? 빠른 조회 버튼
- 월간 소비 리포트(증감률)
- 중복 구매 방지 알림(마트 스캔 시 재고 안내)
- 유통기한 관리 및 만료 임박 알림

---

## 6. 사용자 시나리오
### 시나리오 A: 1인 가구
1. 휴지 12롤 등록
2. 사용할 때마다 1롤 사용 버튼 클릭
3. 앱이 평균 5일 주기로 계산
4. 12일 남았을 때 알림 수신
5. 자동 생성된 장보기 리스트로 미리 구매

### 시나리오 B: 맞벌이 부부
1. 부부가 가족 그룹으로 연결
2. 한 사람이 샴푸 구매 체크
3. 재고가 즉시 반영되어 다른 가족의 중복 구매 방지
4. 월말 리포트로 소비 패턴 확인

---

## 7. 정보 구조와 화면 구성
1. 홈 화면
- 오늘 소진 임박 품목
- 빠른 조회(집에 있어?) 버튼

2. 재고 화면
- 품목 리스트, 현재 수량, 예상 소진일
- 품목 추가 및 수정

3. 사용 기록 화면
- 품목별 사용 버튼
- 최근 사용 이력

4. 장보기 화면
- 자동 생성 리스트
- 수동 추가, 구매 완료 체크

5. 리포트 화면
- 월간 사용량 증감
- 품목별 소비 추세

6. 가족 화면
- 구성원 관리
- 공유 권한 설정

---

## 8. 구현 방식 개요
### 예측 방식
AI 모델 없이도 가능한 규칙 기반 계산으로 시작합니다.

- 평균 사용 간격 = 최근 N회 사용 기록의 평균
- 예상 소진일 = 현재 재고 ÷ 평균 일일 사용량
- 이상치 보정: 과도한 기록은 평균 계산에서 완화 처리

### 초기 MVP 기술 방향
- 모바일 앱 또는 웹앱
- 사용자 계정 기반 동기화
- 알림 시스템(푸시 또는 앱 내 알림)
- 클라우드 DB로 가족 공유 실시간 반영

## 9. 비즈니스 모델
1. 프리미엄 구독
- 고급 리포트, 다중 가구 관리, 알림 커스터마이징

2. 제휴 커머스
- 소진 임박 품목의 최저가 추천, 장보기 연동

3. B2B 확장
- 생활 플랫폼, 커뮤니티 앱, 오피스 비품 관리 솔루션 연계

---

## 10. 성과 지표(KPI)
1. 주간 활성 사용자 수
2. 품목 등록 수와 사용 기록 입력률
3. 소진 전 구매 성공률
4. 중복 구매 감소율
5. 알림 클릭률 및 장보기 전환율
6. 가족 그룹 생성률

---

## 11. 추진 일정(예시)
1. 1단계(1개월): MVP 개발
- 등록, 재고, 사용 기록, 소진 예측, 알림 기본 기능

2. 2단계(2개월): 공유 및 자동화 강화
- 가족 공유, 장보기 자동 생성, 바코드 기능 고도화

3. 3단계(3개월): 데이터 기반 고도화
- 소비 리포트, 추천 구매 시점, 커머스 연동

---

## 12. 기대 효과
1. 생필품 부족 불편 감소
2. 중복 구매 방지로 생활비 절감
3. 장보기 계획화로 시간 절약
4. 가족 간 커뮤니케이션 비용 감소
5. 생활 데이터 기반의 스마트 소비 습관 형성

---

## 13. 공모전 발표용 핵심 문장
- 홈스톡은 재고를 기록하는 앱이 아니라, 생활 패턴을 읽고 필요한 순간을 먼저 알려주는 생활 관리 플랫폼입니다.
- 핵심 경쟁력은 소진 예측, 장보기 자동화, 가족 공유의 연결 구조입니다.
- 사용자는 더 이상 떨어진 뒤에 사지 않고, 미리 알고 효율적으로 소비할 수 있습니다.
